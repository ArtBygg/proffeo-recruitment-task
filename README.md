# Zadanie rekrutacyjne — UI/UX

Fragment systemu **ERP** w **Angular 20** + **Angular Material** + **Tailwind CSS**.

> ⚠️ To jest **wycięty fragment** większej aplikacji. **Większość funkcjonalności nie działa** — brak backendu, logowania, części serwisów i routingu. **Nie naprawiaj tego** i nie przejmuj się błędami w konsoli. Skupiamy się **wyłącznie na warstwie wizualnej i UX**.

## Co masz zrobić

Przejrzyj dwa widoki:

1. **Lista plików** — `src\app\components\task\task-splitview-list`
2. **Szczegóły zadania** — `src\app\components\task\task-splitview-list\task-details-content`

Następnie:

1. **Wymień rzeczy, które Ci nie pasują** z punktu widzenia UI/UX (co jest źle zrobione, niespójne, nieczytelne, niewygodne, niedostępne, źle responsywne itd.).
2. **Zaproponuj zmiany** — krótko, własnymi słowami, na poziomie pomysłów. Bez konieczności przygotowywania mockupów.
3. **Wprowadź wybrane zmiany w kodzie** — które dokładnie, ustalimy wspólnie podczas rozmowy.

## Uruchomienie

```bash
npm install
npm start
```

Aplikacja wystartuje na `http://localhost:4200`. Dane do widoków możesz sobie zamockować lokalnie (statyczna tablica, hardcodowany obiekt) — nie trać czasu na podpinanie backendu.
